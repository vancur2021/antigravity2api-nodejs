# 📝 项目修订记录：参数与提示词控制优化

**修订日期**：2026-03-04
**作者**：萌萌 (Cline Assistant)
**版本状态**：已完成实现并经过 Code Review
**修订说明**：本次修改旨在增强代理服务的灵活性，允许用户决定是否通过后端干预 API 参数及注入预设系统提示词。

---

## 1. 新增功能特性

### 1.1 模型参数自定义开关
*   **功能描述**：在设置页面新增“启用自定义默认值”总开关。
*   **交互逻辑**：
    *   **开启**：当 API 请求中缺失对应参数时，使用页面设置的“温度、Top P/K、最大 Token”作为默认值补全。
    *   **关闭**（默认）：系统不再补全上述四个参数，直接传递请求自带内容。若请求中也缺失，则由上游 API 应用其服务端默认值。
*   **UI 表现**：开关关闭时，对应输入框应用灰度滤镜并禁用编辑，提供清晰的操作反馈。

### 1.2 反代系统提示词控制开关
*   **功能描述**：在“反代系统提示词”文本框上方新增独立“启用”开关。
*   **交互逻辑**：
    *   **开启**：将预设的人设信息（如“萌萌”自我介绍）注入到对话的系统指令中。
    *   **关闭**（默认）：完全跳过预设提示词的注入，使对话环境更纯净，仅保留官方注入和用户请求内容。

---

## 2. 技术实现方案

### 2.1 后端逻辑 (Node.js)
*   **参数规范化 (`src/utils/parameterNormalizer.js`)**：引入 `useCustomDefaultParams` 标记位。当关闭时，参数回退为 `undefined` 以尊重 API 原始设定。
*   **提示词组合 (`src/utils/converters/common.js`)**：在 `buildSystemPromptParts` 中引入状态检查，动态构建消息数组。
*   **配置管理 (`config.json.example`)**：新增 `useCustomDefaultParams` 与 `enableProxySystemPrompt` 字段。

### 2.2 前端界面 (Web UI)
*   **结构更新 (`public/index.html`)**：新增开关容器及 UI 控件。
*   **视觉增强 (`public/style.css`)**：新增 `.disabled` 类样式，利用 `grayscale` 实现禁选反馈。
*   **交互绑定 (`public/js/config.js`)**：实现 UI 属性联动，处理复选框数据流同步。

---

## 3. 注意事项
*   **官方系统提示词**：仍受密码锁保护，严禁随意删除或大幅修改，以防导致 429 频率超限。
*   **配置补全**：旧版升级用户请参考 `config.json.example` 手动更新本地配置字段。
